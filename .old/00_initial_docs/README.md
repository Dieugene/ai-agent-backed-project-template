# 00_initial_docs — Исходные требования

## Назначение
Хранение первоначальных требований, внешних спецификаций, референсных материалов.

## Ответственный
**Архитектор**

## Что хранится

- Исходные требования заказчика
- Инструкции по извлечению данных (extraction instructions)
- Схемы данных (JSON schemas, DB schemas)
- Примеры входных/выходных данных
- Референсная документация внешних систем
- Копии внешних спецификаций

## Правила

1. **Файлы НЕ изменяются в процессе разработки**
   - Это "только для чтения" источник первоначальных требований
   - Сохраняем исторический контекст

2. **Актуальная спецификация:**
   - Ведется в `00_docs/project_spec.md`
   - При расхождении с исходными требованиями — приоритет у актуальной спецификации

3. **Формат файлов:**
   - Любой (Markdown, PDF, JSON, XLSX и т.д.)
   - Желательно добавить README с описанием содержимого

## Примеры файлов

- `requirements_v1.pdf` — исходные требования заказчика
- `data_schema.json` — схема входных данных
- `extraction_instructions.md` — правила извлечения информации
- `reference_examples/` — примеры входных/выходных данных
