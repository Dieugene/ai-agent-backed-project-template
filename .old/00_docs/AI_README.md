## Роли и ответственность

### Архитектор / Постановщик
- Определяет цели, утверждает структуру каталогов, форматы данных
- Приоритизирует итерации и принимает архитектурные решения
- **НЕ пишет код** — занимается только постановкой задач и валидацией результатов

### Разработчик
- Реализует функционал согласно постановке задачи
- Поддерживает документацию и окружение
- Фиксирует результаты в разделе "Отчёт разработчика" в файлах итераций

**Детали ролей и порядок взаимодействия:** см. `01_agents/roles.md`

> **Важно:** Архитектор не пишет код и не изменяет исходники. Его зона ответственности — постановка задач и валидация результатов.

### Структура каталогов

| Путь | Назначение | Ответственный | Что кладём |
|------|------------|---------------|------------|
| `00_docs/` | Документация, спецификации, инструкции | оба агента | AI_README.md, policies.md, project_spec.md |
| `00_initial_docs/` | Исходные требования и референсы (опционально) | архитектор | исходные требования, схемы данных, примеры |

## Общий поток задач

1. **Формулировка** (архитектор):
   - Обновляет требования в `00_docs/` при необходимости
   - Создает папку `02_tasks/NNN_task_name/`
   - Пишет файл `iteration_NNN.md` с полной постановкой

2. **Регистрация** (архитектор):
   - Заносит задачу в раздел "Состояние задач" ниже
   - Устанавливает статус "в работе"

3. **Исполнение** (разработчик):
   - Читает постановку из `iteration_NNN.md`
   - Обновляет соответствующие файлы/папки
   - Фиксирует результаты в `04_data/`, `05_logs/` или `02_tasks/NNN_task_name/artifacts/`

4. **Отчет** (разработчик):
   - Добавляет раздел "Отчёт разработчика" в конец `iteration_NNN.md`
   - Уведомляет архитектора о завершении

5. **Ревизия** (архитектор):
   - Проверяет результаты
   - Обновляет статус задачи на "выполнена"
   - Добавляет новые требования при необходимости

## Конвенции и соглашения

### Нумерация
- Префиксы папок: `00_`, `01_`, `02_` и т.д. — отражают логический порядок
- Задачи и модули: трехзначная нумерация `001_`, `002_`, `003_`
- Timestamped папки: формат `YYYYMMDD_HHMMSS` (например, `run_20251216_143000`)

### Окружение Python (если применимо)
- Создание окружения: `python -m venv .venv`
- Активация:
  - Windows: `.venv\Scripts\activate`
  - Linux/Mac: `source .venv/bin/activate`
- Установка зависимостей: `pip install -r requirements.txt`
- Секреты и API ключи в `.env` файле (не коммитится в Git)

### Окружение Node.js (если применимо)
- Установка зависимостей: `npm install` или `yarn install`
- Секреты в `.env` файле или через `dotenv`

### Docker (если применимо)
- Локальный запуск: `docker-compose up --build`
- Переменные окружения задаются в `docker-compose.yml` или `.env`
