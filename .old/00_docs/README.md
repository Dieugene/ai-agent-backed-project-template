# 00_docs — Документация проекта

## Назначение
Центральное хранилище всей проектной документации, спецификаций и инструкций.

## Ответственный
Оба агента:
- **Архитектор** создает и обновляет спецификации, требования
- **Разработчик** поддерживает техническую документацию

## Что хранится

### Обязательные файлы
- **`AI_README.md`** — стартовая точка для любого агента, центральный документ проекта
  - Описание проекта
  - Роли и ссылки на `roles.md` и `policies.md`
  - Структура каталогов
  - Состояние задач
  - Стартовые промпты
  - Конвенции и окружение

- **`policies.md`** — политики работы с проектом
  - Что хранится в каждой папке
  - Правила формирования промптов
  - Workflow работы над задачами
  - Конвенции окружения

### Опциональные файлы
- **`project_spec.md`** — техническая спецификация проекта
- **`api_docs.md`** — документация API
- **`architecture.md`** — описание архитектуры
- Другие технические документы по необходимости

## Правила

1. **НЕ складываем сюда:**
   - Данные (это в `04_data/`)
   - Логи (это в `05_logs/`)
   - Исходный код (это в `03_src/`)

2. **Документация:**
   - Пишется на русском или английском языке (согласно выбору проекта)
   - Каждый документ имеет четкое назначение
   - Используется Markdown формат

3. **Обязательная ссылка:**
   - `AI_README.md` ОБЯЗАН ссылаться на `policies.md` в разделе "Перед началом работы"
   - Каждый агент ДОЛЖЕН прочитать оба файла перед началом работы
