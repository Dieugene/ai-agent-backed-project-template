# 02_tasks — Итерации и задачи разработки

## Назначение
Хранение постановок задач, итераций разработки и отчетов о выполнении.

## Ответственные
- **Архитектор** — создает папку задачи и файл постановки
- **Разработчик** — добавляет раздел "Отчёт разработчика"

## Структура

```
02_tasks/
├── 001_task_name/
│   ├── iteration_001.md        # Постановка + Отчет
│   └── artifacts/               # Артефакты задачи (опционально)
│       └── screenshot.png
├── 002_another_task/
│   ├── iteration_002.md
│   └── artifacts/
├── 003_third_task/
│   └── iteration_003.md
└── README.md
```

## Правила именования

### Папки задач
- **Формат:** `NNN_task_name/`
- **Нумерация:** строго трехзначная, последовательная (001, 002, 003, ...)
- **Название:** краткое отражение сути задачи
- **Стиль:** латиница, snake_case, без пробелов
- **Примеры:**
  - `001_setup_environment`
  - `002_pdf_processing`
  - `003_api_integration`

### Файлы итераций
- **Формат:** `iteration_NNN.md`
- **Нумерация:** совпадает с номером папки
- **Примеры:**
  - `iteration_001.md`
  - `iteration_002.md`

## Содержимое iteration_NNN.md

### Часть 1: Постановка (пишет Архитектор)
1. **Цель** — что мы хотим достичь
2. **Контекст** — ссылки, зависимости, что уже есть
3. **Задачи** — детальный список действий (пошаговый)
4. **Технические требования** — окружение, архитектура, данные
5. **Артефакты** — что будет создано/изменено
6. **Критерии завершения (DoD)** — когда задача считается выполненной
7. **Примечания и ограничения** — важная информация для разработчика
8. **Зависимости** — от каких задач зависит, что блокирует

### Часть 2: Отчёт разработчика (добавляет Разработчик)
1. **Выполненные работы** — краткое описание
2. **Созданные/измененные файлы** — список с путями
3. **Артефакты** — пути к данным, логам, скриншотам
4. **Результаты тестов/проверок** — описание проверок
5. **Инструкция по воспроизведению** — как повторить результат
6. **Вопросы и риски** — что требует внимания архитектора
7. **Предложения по улучшению** — идеи для будущих итераций

## Папка artifacts/

### Назначение
Хранение специфических артефактов задачи, которые не подходят для `04_data/` или `05_logs/`.

### Что хранить
- Скриншоты результатов
- Промежуточные визуализации
- Небольшие примеры данных для демонстрации
- Диаграммы, схемы специфичные для задачи

### Что НЕ хранить
- Глобальные данные проекта → `04_data/`
- Логи запусков → `05_logs/`
- Код → `03_src/`
- Большие файлы данных

## Workflow работы с задачами

### 1. Создание задачи (Архитектор)
```bash
# Создать папку
mkdir 02_tasks/NNN_task_name
mkdir 02_tasks/NNN_task_name/artifacts

# Создать файл (скопировать шаблон)
cp 02_tasks/001_template_task/iteration_001.md \
   02_tasks/NNN_task_name/iteration_NNN.md

# Заполнить постановку
# vim 02_tasks/NNN_task_name/iteration_NNN.md
```

### 2. Регистрация в AI_README.md (Архитектор)
- Добавить задачу в таблицу "Состояние задач"
- Установить статус "в работе"

### 3. Передача разработчику (Архитектор)
Короткий промпт:
```
Ты выступаешь в роли разработчика (см. 01_agents/roles.md).

Перед началом прочитай:
- 01_agents/roles.md
- 00_docs/AI_README.md
- 00_docs/policies.md
- 02_tasks/NNN_task_name/iteration_NNN.md

Задача: Выполнить задачу, описанную в iteration_NNN.md

После выполнения сообщи постановщику.
```

### 4. Выполнение (Разработчик)
- Читает постановку
- Реализует функционал
- Сохраняет артефакты

### 5. Отчет (Разработчик)
- Добавляет раздел "Отчёт разработчика" в конец `iteration_NNN.md`
- Уведомляет архитектора

### 6. Приемка (Архитектор)
- Проверяет результаты
- Обновляет статус в `AI_README.md` на "выполнена"

## Важно

1. **Постановка содержит ВСЕ детали**
   - Промпт агенту НЕ дублирует детали
   - Все конкретные действия в `iteration_NNN.md`

2. **Разработчик ОБЯЗАН добавить отчет**
   - Без отчета задача не считается завершенной
   - Отчет добавляется в конец того же файла

3. **Нумерация строго последовательная**
   - Нельзя пропускать номера
   - При удалении задачи — не переиспользовать номер
