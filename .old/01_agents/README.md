# 01_agents — Роли агентов

## Назначение
Описание ролей, ответственности и правил взаимодействия агентов.

## Ответственный
**Архитектор**

## Что хранится

### Обязательные файлы
- **`roles.md`** — описание всех ролей проекта
  - Архитектор / Постановщик
  - Разработчик
  - Валидатор / Ревьюер (опционально)
  - Зоны ответственности каждой роли
  - Ограничения (что НЕ делает роль)
  - Workflow взаимодействия между ролями

### Опциональные файлы
- `architect_chat_history.md` — экспорт диалогов архитектора (для анализа)
- `developer_chat_history.md` — экспорт диалогов разработчика (для анализа)
- Дополнительные правила взаимодействия

## Правила

1. **Каждая роль должна иметь:**
   - Четкое описание зоны ответственности
   - Явные ограничения (что НЕ делает)
   - Ключевые действия
   - Пример workflow

2. **НЕ хранить здесь:**
   - Код (это в `03_src/`)
   - Данные (это в `04_data/`)
   - Постановки задач (это в `02_tasks/`)

3. **Обновление:**
   - При изменении процессов взаимодействия — обновляется `roles.md`
   - Архитектор согласует изменения перед внедрением

## Обязательная структура roles.md

1. Описание каждой роли
2. Зоны ответственности
3. Ограничения
4. Ключевые действия
5. Взаимодействие между ролями (workflow)
6. Правила эскалации вопросов
